Detailed Description

A Connection is what actually sends (or simulates sending of) a Message.

A Connection is really a half-connection; it has a pointer to it's other half

When 'send' is called, it forwards the message to other half. When receives incomingMessage from other half, forwards to Message handler.

Generally a PseudoTcpTx object will call 'send', while a PseudoTcpRx object will handle incoming messages.

The 'send' part can simulate network problems by losing, delaying, or duplicating messages.

id An id for the connection, can be useful in debugging.
